## üîß Technical Details

### Browser Requirements
- Modern browser (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
- WebRTC support (for WHIP protocol)
- MediaStream API support
- IndexedDB support (for offline storage)

### Streaming Protocols

**RTMP/RTMPS** - Most compatible
- Best for: Twitch, YouTube, Facebook
- Latency: 3-8 seconds
- Browser support: Via preview mode

**SRT** - Low latency
- Best for: Professional broadcasting
- Latency: <1 second
- Browser support: Via WebTransport proxy

**WebRTC-WHIP** - Ultra-low latency
- Best for: Interactive streaming
- Latency: <500ms
- Browser support: Native (Chrome 97+)

**HLS** - Receive-only
- Best for: Playback
- Latency: 6-30 seconds
- Browser support: Native (Safari) or HLS.js

### How It Works

1. **Media Capture**: Browser's `getUserMedia()` captures video/audio
2. **Preview**: MediaStream displayed in local `<video>` element
3. **Protocol Selection**: Auto-detects protocol from URL
4. **Connection**: Establishes connection to streaming server
5. **Transmission**: Sends encoded media to destination
6. **Monitoring**: Tracks stats, handles reconnections

## üî• Pro Tips

### Better Quality
